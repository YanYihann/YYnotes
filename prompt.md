# 一、核心目标

把我提供的原始笔记内容整理成一份高质量的双语 MDX 笔记，要求：

1. **中英文双语逐句对应**
   - 必须采用“英文一行，中文紧跟下一行”的格式。
   - 不允许先整段英文、再整段中文。
   - 必须保证英文与中文一一对应，方便对照阅读和记忆。

2. **结构清晰，有层级感**
   - 使用明确的标题层级（#、##、###）。
   - 各部分逻辑清楚，适合复习、考试、预习和后续补充。
   - 对零散内容进行合理归类、整合和重组。

3. **内容完整，不遗漏重点**
   - 不遗漏原始笔记中的核心概念、定义、步骤、结论、公式、例子、对比、老师强调内容、注意事项等。
   - 如果原笔记内容很碎片化，可以在**不改变原意**的前提下整理成完整表述。

4. **重点突出**
   - 重要内容要适当使用：
     - 粗体：`**重点**`
     - 红字：`<span style="color:red">特别重要内容</span>`
   - 红字用于特别关键、特别易考、特别容易混淆的内容，不要滥用。

5. **适合 MDX/Markdown**
   - 全文输出为 MDX/Markdown 兼容格式。
   - 排版整齐统一，适合直接保存为 `.mdx` 文件。

6. **补充内容需明确标注**
   - 若为了帮助理解而补充了原笔记之外的说明、例子、背景知识，必须标注：**【补充内容】**。
   - 不要把补充内容和原始笔记内容混在一起而不加说明。

# 四、内容整理原则

1. **严格依据原始笔记**
   - 以我提供的内容为主
   - 不要随意脱离原文发挥
   - 不要编造不存在的信息

2. **可以合理优化表达**
   - 可将零散关键词整理成完整句
   - 可将跳跃内容补足逻辑衔接
   - 可将重复内容适度合并

4. **补充内容必须标注**

要求：

# 十五、数学公式生成要求（新增，必须严格遵守）

凡是涉及数学内容（公式、等式、推导、符号表达、函数关系），必须使用**标准 LaTeX 数学公式语法**输出，确保可被 KaTeX/Markdown 正确渲染。

1. 行内公式使用：`$...$`
   - 例如：`$f(x)=x^2+1$`

2. 独立公式使用：`$$...$$`
   - 例如：

```md
$$
f(x)=P+f[pq](x-p)+f[pqr](x-p)(x-q)
$$
```

3. 禁止纯文本公式
   - 不要输出：`f(x) = P + f[pq](x - p) + ...`（纯文本）
   - 必须输出为可渲染公式（`$...$` 或 `$$...$$`）

4. 常见公式语法规范
   - 分式使用 `\frac{a}{b}`
   - 上下标使用 `x_i`、`x^{n}`
   - 根号使用 `\sqrt{...}`
   - 求和/积分使用 `\sum`、`\int`
   - 必要时使用 `\left( ... \right)` 保证括号匹配

5. 多行推导请使用对齐环境

```md
$$
\begin{aligned}
a &= b + c \\
  &= d + e
\end{aligned}
$$
```

6. 双语规则不变
   - 解释文字继续保持“英文一行 + 中文一行”
   - 但数学公式本体不需要逐字翻译，可直接复用同一公式块

# 十六、一致性与稳定输出要求（新增，必须严格遵守）

为保证每次生成的笔记风格、结构与表达一致，请严格执行以下规则：

1. 同输入同输出风格
   - 在输入材料相同或高度相似时，输出的章节顺序、标题命名、术语译法、表达风格、列表层级必须保持一致。
   - 不要因措辞偏好随意改写同一知识点的表达方式。

2. 固定章节骨架（通用版，按内容适配）
   - `## 学习目标 / Learning Objectives`
   - `## 背景与上下文 / Background and Context`（可选）
   - `## 关键概念 / Key Concepts`
   - `## 方法、流程或机制 / Methods, Process, or Mechanism`
   - `## 证据、数据或材料依据 / Evidence, Data, or Supporting Materials`（可选）
   - `## 案例与应用 / Examples and Applications`
   - `## 易错点、边界与注意事项 / Pitfalls, Limits, and Notes`
   - `## 小结 / Summary`
   - `## 练习题或行动项 / Practice Questions or Action Items`
   - 说明：不得强制套用某一学科专属结构；若是数学、代码、实验类内容，再按需要增加对应专题小节。

3. 术语映射必须稳定
   - 同一术语在同一份笔记内只能使用一种主译法。
   - 若首处采用“中文（English）”形式，后续保持同一写法，不得频繁切换同义词。

4. 专业记号必须稳定（按学科适配）
   - 若存在公式、代码、符号、缩写、变量名、实验名词，全文保持一致。
   - 同一对象在不同段落重复时，保持相同写法，不随意替换等价但不同形式。

5. 列表与编号风格固定
   - 同级内容统一使用同一种列表风格（有序或无序），避免来回切换。
   - 步骤型内容默认使用有序列表，特征/要点默认使用无序列表。

6. 禁止随机化表达
   - 不使用“可替换但不必要”的随机句式、随机标题、随机总结模板。
   - 不添加与原文无关的扩展段落来改变篇幅结构。

7. 信息不足时保持保守
   - 对原文缺失信息，不要臆造细节；如需补充，必须标注 **【补充内容】**。
   - 若存在不确定表述，优先使用中性、可验证写法，避免强断言。

8. 生成前自检一致性（隐式执行，不要输出该检查过程）
   - 检查章节顺序是否与通用骨架一致且合理裁剪。
   - 检查术语译法、专有名词、记号是否前后一致。
   - 检查是否满足“英文一行 + 中文一行”的双语对应规则。
   - 检查 Markdown/MDX 结构是否完整、可渲染、可复用。
