  # 三、结构整理规则
  
  1. **标题整理**
  
     - 如果原文已经有明显标题，就保留并规范层级。
     - 如果原文没有清晰标题，可以根据内容自然分段拟定简洁小标题，但不要改变原文内容本身。
     - 所有标题都必须补上规范编号，如“1 ”“1.1”“1.1.1”等。
     - 标题格式统一为：**编号 + 中文标题 + 空格 + 英文标题**，并且中英文标题同一行显示。
  
  2. **段落整理**
  
     - 将相关内容放在同一小节下。
     - 同一主题内部保持紧凑，不要过度拆分。
     - 一段太长时可以合理分段，但不能删减句子。
     - 在不改变原文内容的前提下，优先把长段内容拆解到列表项中。
     - 正文内容必须遵循：**中文一行，英文下一行，逐行对应。**
     - 中文与英文之间使用 **Markdown 行内换行**，即上一行末尾保留两个空格，形成 `Shift+Enter` 的换行效果。
  
  3. **列表整理**
  
     - 原文中如果是并列概念、特点、步骤、例子、条件、结论等，优先整理成项目符号或编号列表。
  
     - 列表内容必须保留原话，不要改写成另一种表达。
  
     - 若原文逻辑上有先后顺序，使用**有序列表**。
  
     - 若原文逻辑上是并列展开，使用**无序列表**。
  
     - 每一小节至少应尽量包含一组列表，除非内容客观上完全不适合列表化。
  
     - **列表排版格式必须如下：**
  
       - 中文行使用 `-` 或 `1.` 等列表标记。
       - 中文下一行对应英文**不再重复** `-` 或 `1.`。
       - 英文行作为该列表项内容的下一行，用缩进配合 Markdown 行内换行保持对应关系。
  
     - 示例格式：
  
       ```markdown
       - 中文内容  
         English content
       
       1. 中文内容  
          English content
       ```
  
  4. **表格整理**
  
     - **如果原文中存在比较关系、对照关系、分类对比、方法区别、概念区分等内容，应优先整理成表格。**
     - 仅当原文中存在明显可对照的内容时，才可整理成表格。
     - 表格中的文字也必须尽量保持原文表达。
     - 表格若采用双语排版，也需保持对应关系清晰，不得改写原意。
  
  5. **图片内容整理**

     - 图片中的文字内容应按照其原有逻辑关系重组到合适的小节中。
     - 图片中的对照关系适合用表格时，整理成表格；适合用列表时，整理成列表。
     - 如果图片中某些内容因排版原因呈现为上下、左右、分栏、框图等结构，应尽量在 Markdown 中还原其逻辑层级。
     - 不要忽略图片中的小字说明、标注、坐标标签、页边注释等可辨识文字内容。
     - 对图片内容的处理仍然遵循：**只整理格式，不改写原文。**
  
  6. **重点标注整理**
  
     - 如果原文中有天然的重点层次，例如标题高亮、加粗词、关键词、结论句、定义句、反复出现的核心概念，可以通过深红色 HTML 进行强调。
     - 优先使用：
       - `<font color='#8B0000'>重点内容</font>`
     - 特别重要时可使用：
       - `<font color='#8B0000'>**特别重要内容**</font>`
     - 不要把整段大面积标红，应尽量只标核心词、核心短语、核心句。
     - 重点标注必须服务于原文结构，不得制造原文没有的“新重点”。
  
  7. **引用块总结**

       > 比如说，……

     - 总结要通俗，但不能脱离原文内容乱发挥。
  
     - 引用块总结也必须遵循**一行中文、一行英文**的形式。
  
     - 引用块内部中英文之间也使用 **Markdown 行内换行**，即 `Shift+Enter` 的效果。
